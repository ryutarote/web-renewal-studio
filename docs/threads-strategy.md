# Threads AI自動化運用戦略

## 概要

Claude Codeを活用した6エージェント構成のThreads自動運用システム。
「5%人間介入モデル」を採用し、完全自動の安定収益と人間の審美眼による爆発力を両立する。

---

## 1. システムアーキテクチャ

### エージェント構成（6体）

```
リサーチャー → ライター → ポスター → フェッチャー → アナリスト
     ↑                                                    |
     └────────────── フィードバックループ ──────────────────┘
```

| エージェント | 役割 | 自動/手動 |
|---|---|---|
| リサーチャー | YouTube・X・Instagram巡回、ネタ収集 | 完全自動 |
| ライター | 投稿生成（15パターン＋品質採点） | 自動＋人間チェック |
| ポスター | API経由で時間帯分散投稿 | 完全自動 |
| フェッチャー | 投稿メトリクス自動取得 | 完全自動 |
| アナリスト | パフォーマンス分析・改善提案 | 完全自動 |
| コーディネーター | 全体フロー制御 | 完全自動 |

### 運用モード

| モード | 人間介入 | 想定月収 | 向いている人 |
|---|---|---|---|
| 完全自動（0%） | なし | 3〜5万円/アカウント | 複数アカウント並行運用、副業で時間がない人 |
| 5%人間介入 | 下書きチェックのみ（1日10分） | 数十万円〜 | 1アカウントを本気で伸ばしたい人 |

## 2. アカウント設計

### 2.1 ナレッジファイル構成

新規アカウント追加時に必要なファイル：

```
accounts/
└── {account_name}/
    └── knowledge/
        ├── profile.md          # 口調・キャラ設定
        ├── target.md           # ターゲットペルソナ
        ├── genre.md            # ジャンル情報
        ├── ng-words.md         # NGワード・禁止表現
        ├── strategy.md         # テーマツリー・戦略
        └── domain-knowledge.md # ジャンル専門知識
```

### 2.2 プロフィール設計（profile.md）

定義すべき項目：

- **キャラクター名**：アカウントの人格
- **口調**：ですます調 / タメ口 / 辛口 / 親しみ系 など
- **一人称**：僕 / 私 / 俺 など
- **語尾の癖**：特徴的な表現パターン
- **立場・ポジション**：実践者 / 研究者 / 教える人 / 共感する人
- **価値観**：何を大事にしているか
- **禁止事項**：絶対にやらないこと・言わないこと

### 2.3 ターゲット設計（target.md）

- **メインターゲット**：年齢・性別・職業・課題
- **サブターゲット**：拡張層
- **ペインポイント**：ターゲットが抱える悩み（5つ以上）
- **理想の状態**：ターゲットがなりたい姿
- **情報収集行動**：普段どこで情報を得ているか

### 2.4 テーマツリー設計（strategy.md）

```
{ジャンル名} テーマツリー
├── カテゴリA
│   ├── サブテーマA-1
│   ├── サブテーマA-2
│   └── サブテーマA-3
├── カテゴリB
│   ├── サブテーマB-1
│   ├── サブテーマB-2
│   └── サブテーマB-3
├── カテゴリC
│   ├── サブテーマC-1
│   ├── サブテーマC-2
│   └── サブテーマC-3
└── カテゴリD
    ├── サブテーマD-1
    ├── サブテーマD-2
    └── サブテーマD-3
```

## 3. 投稿作成ルール

### 3.1 1行目（フック）ルール

- バズ投稿265件以上のパターン集から構造を学習
- ジャンルに合わせて変換して生成
- **断言系**・**問いかけ系**・**暴露系**・**数字系**・**共感系** などの型を使い分け

### 3.2 投稿パターン（15種類以上）

| # | パターン名 | 特徴 |
|---|---|---|
| 1 | 短文完結型 | 1〜3行で完結。インパクト勝負 |
| 2 | コメント誘導型 | 質問・投げかけで返信を狙う |
| 3 | ツリー型 | 本文→返信で詳細展開 |
| 4 | 暴露系 | 業界の裏側・知られざる事実 |
| 5 | リスト系 | 箇条書きで読みやすく |
| 6 | ストーリー系 | 体験談・失敗談 |
| 7 | 比較系 | A vs B で議論を生む |
| 8 | How-to系 | 具体的な手順・方法 |
| 9 | 逆張り系 | 常識の逆を突く |
| 10 | データ系 | 数字・統計で説得力 |
| 11 | トレンド便乗型 | 今話題のネタに乗る |
| 12 | 共感型 | 「わかる」を引き出す |
| 13 | 煽り型 | 危機感・焦りを刺激 |
| 14 | 教訓型 | 学びを1文にまとめる |
| 15 | 実績公開型 | 数字で成果を見せる |

### 3.3 自動ローテーション

- 直近3件と同じパターンは使用禁止
- 自動で別パターンに切り替え

### 3.4 品質チェック（10項目採点）

| # | 項目 | 評価基準 |
|---|---|---|
| 1 | フックの強さ | 1行目で止まるか |
| 2 | 有益性 | 読者にとって価値があるか |
| 3 | 具体性 | 抽象的でなく具体的か |
| 4 | テンポ感 | リズムよく読めるか |
| 5 | 共感性 | 「わかる」と思えるか |
| 6 | 独自性 | 他と差別化できているか |
| 7 | CTA強度 | 行動を促せるか |
| 8 | ターゲット適合 | ペルソナに刺さるか |
| 9 | 文字数適正 | 長すぎ・短すぎでないか |
| 10 | 炎上リスク | 問題表現がないか |

### 3.5 品質フィルター

- **平均7.0未満** → 自動棄却（書き直し2回でダメなら破棄）
- **過去100件と類似度0.85以上** → 自動棄却（同じ投稿ループ防止）

## 4. 投稿スケジュール

### 4.1 投稿頻度

- 1日5〜10本（5%人間介入モード）
- 1日10本（完全自動モード）
- 約2時間おきに1本ずつ時間帯を分散

### 4.2 推奨投稿時間帯

| 時間帯 | 優先度 | 理由 |
|---|---|---|
| 7:00-8:00 | ★★★ | 通勤時間帯 |
| 12:00-13:00 | ★★★ | 昼休み |
| 18:00-19:00 | ★★★ | 退勤・帰宅時間 |
| 21:00-22:00 | ★★★ | ゴールデンタイム |
| 9:00-11:00 | ★★ | 午前の隙間時間 |
| 15:00-17:00 | ★★ | 午後の隙間時間 |
| 23:00-0:00 | ★ | 深夜帯（特定層向け） |

## 5. メトリクス計測

### 5.1 計測タイミング

| タイミング | 目的 |
|---|---|
| 投稿1時間後 | 初速チェック（バズの兆候確認） |
| 投稿6時間後 | 中間計測（伸び方の判断） |
| 投稿24時間後 | 最終計測（確定値） |

### 5.2 取得データ

- 閲覧数（Views）
- いいね数（Likes）
- リプライ数（Replies）
- 保存数（Saves）
- エンゲージメント率 = (いいね + リプライ + 保存) / 閲覧数

## 6. 分析・改善サイクル

### 6.1 アナリストの分析項目

- **パターン別パフォーマンス**：どの投稿パターンが伸びているか
- **テーマ別パフォーマンス**：どのテーマに反応があるか
- **1行目の型別分析**：どのフック型が効果的か
- **時間帯別分析**：いつ投稿すると伸びるか
- **トレンド変化**：反応が上がっている/下がっているテーマ

### 6.2 フィードバック先

| 分析結果 | フィードバック先 | アクション |
|---|---|---|
| 「断言系1行目が伸びている」 | ライター | 断言系を多めに生成 |
| 「テーマXが飽きられている」 | リサーチャー | 別テーマを重点調査 |
| 「リスト系が保存数高い」 | ライター | リスト系の比率を上げる |
| 「21時台の投稿が強い」 | ポスター | 21時台に良い投稿を配置 |

## 7. マネタイズ戦略

### 7.1 収益化フロー

```
Threads投稿 → フォロワー増加 → リスト獲得 → 商品販売
                                    ↓
                              CTA設計が重要
```

### 7.2 マネタイズ手段

| 手段 | 難易度 | 収益性 | 備考 |
|---|---|---|---|
| ASPアフィリエイト | ★ | 月3〜5万 | 完全自動でも可能 |
| note販売 | ★★ | 数十万〜 | リスト獲得後に展開 |
| コンサル | ★★★ | 高単価 | 実績が必要 |
| 自社商品 | ★★★ | スケーラブル | 設計に時間がかかる |

### 7.3 リスト獲得戦略

- プロフィールリンクにLP/オプトインページを設置
- 投稿内で無料特典への誘導
- 目標：4日で700リスト（5%人間介入モデル実績）


## 9. 注意事項・リスク管理

### 9.1 運用上の注意

- **ナレッジ設計が最重要**：ツールだけ入れても稼げない。設計が先。
- **類似度チェック必須**：同じ投稿のループを防止する
- **NGワード管理**：ジャンルごとに炎上リスクのある表現を定義
- **API制限の把握**：Threads APIのレートリミットを把握して投稿間隔を調整

### 9.2 品質管理

- 品質スコア7.0以上のみ投稿（自動フィルター）
- 過去100件との類似度0.85以上は棄却
- 直近3件と同じパターンは使用禁止
- 5%人間チェックで「今出すべき投稿」を選別

### 9.3 コスト

- Claude Code サブスクリプション：月$100〜$200
- 追加のAPIコスト：Threads API（無料）
